# Landing Page Prompt

你是资深产品设计师 + 前端工程师。请构建一个**简约、美观、专业**的 SaaS 落地页，支持**中文/英文一键切换**（i18n）。

## 目标
- 为产品创建高转化首页，突出价值、可信度与行动引导（CTA）。
- 页面在桌面端和移动端都具备优秀体验。
- 默认语言跟随浏览器语言（`zh` 或 `en`），并允许用户手动切换。

## 风格要求
- 视觉关键词：Minimal / Clean / Professional / Trustworthy。
- 留白充足，排版克制，避免花哨装饰。
- 使用统一设计系统（颜色、字体、间距、圆角、阴影、动效节奏）。
- 颜色建议：中性底色 + 1 个品牌主色 + 1 个强调色。
- 动效轻量（如淡入、上浮、微交互），不影响性能和可读性。

## 页面结构
1. Header
- Logo、导航、语言切换（中/EN）、主 CTA 按钮。
- 吸顶但不喧宾夺主。

2. Hero
- 清晰主标题 + 副标题（中英文本都要提供）。
- 主要 CTA 与次级 CTA。
- 右侧可放产品界面 Mockup 或价值可视化图形。

3. Value Props
- 3-4 个核心价值点（图标 + 标题 + 简述）。

4. Social Proof
- 客户 Logo 墙或数据指标（如用户数、效率提升百分比）。
- 1-2 条简短用户评价。

5. Feature Section
- 2-3 组功能介绍（标题、说明、示意图/占位图）。
- 支持左右交错布局，提高可读性。

6. Pricing/CTA Block
- 简洁套餐或行动召唤模块。
- 强调“立即开始/免费试用/预约演示”。

7. FAQ
- 4-6 个常见问题，支持中英双语内容。

8. Footer
- 基础链接、版权、联系方式、社媒入口。

## 多语言与文案规范
- 所有用户可见文案必须通过 i18n 字典管理（不要在组件中硬编码）。
- 至少包含 `zh-CN` 和 `en-US` 两套文案。
- 语言切换后即时生效，不刷新页面。
- 数字、日期、货币格式可按 locale 适配。

## 交付要求
- 输出可运行代码（组件化、结构清晰、可维护）。
- 响应式：覆盖移动端（>=360px）、平板、桌面。
- 可访问性：语义化标签、键盘可达、对比度达标、图片含 alt。
- 性能：首屏内容优先，图片懒加载，避免过重依赖。
- SEO：包含 `title`、`description`、Open Graph 基础信息。

## 技术偏好（可按项目调整）
- React + TypeScript
- Tailwind CSS（或等价原子化方案）
- i18n：`react-i18next` 或 `next-intl`
- 动效：`framer-motion`（仅必要场景）

## 输出格式
请按以下顺序输出：
1. 页面信息架构（IA）简述
2. 设计令牌（颜色、字体、间距、圆角）
3. 完整页面代码（含 i18n 配置与字典）
4. 如何运行与二次修改说明
5. 可选优化建议（A/B 测试点）
